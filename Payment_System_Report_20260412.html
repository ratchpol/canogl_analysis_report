📋 1. ภาพรวมระบบ

ระบบ Payment Gateway รองรับการชำระเงินผ่าน QR Code หลายช่องทาง เชื่อมต่อกับธนาคารและ e-Wallet ในประเทศไทย

Flow การทำงาน Thai QR (TTB)

Flow A — สร้าง QR & Polling:
สร้าง QR → เพิ่มเข้าระบบ → Poll สถานะ TTB → TTB ตอบกลับ

Flow B — Bank Notify (Push):
TTB ส่ง Notify → บันทึก DB → แจ้ง Merchant → LINE แจ้งลูกค้า

5.3 ปัญหาการเชื่อมต่อ Provider ภายนอก

🟠 TrueMoney Wallet — API Error (55 รายการ)
  • 400 Bad Request (38 รายการ) — เกิดช่วง 20:57 น. อาจเป็นปัญหาชั่วคราวฝั่ง TrueMoney
  • Payment code expired — ลูกค้าใช้ code หมดอายุ
  • Insufficient fund — ยอดเงินไม่พอ

💡 แนวทาง: ตรวจสอบกับ TrueMoney ว่ามี API change หรือไม่

🟠 Alipay+ — Notify Error (666 รายการ)
  • แจ้งผลลูกค้าไม่สำเร็จ (646 รายการ) — เกิดทุกนาทีตลอด 3 ชั่วโมง
  • ไม่พบ paymentrequestid + NullReference (20 รายการ)

💡 แนวทาง: ตรวจสอบ endpoint notify ลูกค้า Alipay+ และแก้ NullReference

🟠 CIL — Merchant Notify Error (1,255 รายการ)
  • ไม่พบ merchant profile (836 รายการ) — merchant 200015, 200047
  • POST callback ล้มเหลว (414 รายการ)
  • NullReferenceException (5 รายการ)

💡 แนวทาง: ตรวจสอบ merchant profile ที่หายไป และ URL callback

🔵 ShopeePay — Query Error (9 รายการ)

ไม่พบ transaction เมื่อ query สถานะ — อาจเป็นเพราะ transaction ยังไม่ถูกสร้างฝั่ง ShopeePay

🔵 Token Error (10 รายการ)

Token ไม่ valid สำหรับ order query — ควรตรวจสอบ token refresh mechanism

📌 7. สรุปผลการวิเคราะห์

✅ จุดแข็ง

  • ความเร็วประมวลผลดีมาก — 98.2% จบใน sub-second
  • ไม่มี transaction ใดเกิน 1 วินาที
  • TTB API ตอบกลับเร็ว — เฉลี่ย 0.48 วินาที
  • รองรับ peak 6 tx/sec ได้ไม่มีปัญหา
  • รองรับหลาย payment provider พร้อมกัน
  • มีระบบ LINE notify แจ้งลูกค้าอัตโนมัติ

⚠️ จุดที่ต้องปรับปรุง

  • DB Error 1,858 รายการ (duplicate + truncation)
  • Schema ไม่ตรงใน savefiletransaction
  • CIL notify ล้มเหลว 1,255 รายการ
  • Alipay+ notify ลูกค้าล้มเหลว 646 รายการ
  • Order Query poll ถี่เกินไป (98.4% ไม่พบ)
  • TrueMoney API error ช่วง 20:57 น.
📌 สรุปภาพรวม:

ระบบ Payment Gateway ทำงานได้ดีในภาพรวม ความเร็วประมวลผลอยู่ในเกณฑ์ดีมาก ปัญหาหลักที่ต้องแก้ไขเร่งด่วนคือ Database errors (duplicate key, data truncation, schema mismatch) ซึ่งกระทบต่อการบันทึกธุรกรรมประมาณ 1,868 รายการจากทั้งหมด 7,200 รายการ (25.9%) นอกจากนี้ควรปรับปรุง notification ไปยัง merchant และลูกค้าที่ยังล้มเหลวอยู่
