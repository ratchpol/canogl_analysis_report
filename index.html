1. ภาพรวมระบบ

ระบบ Payment Gateway รองรับการชำระเงินผ่าน QR Code หลายช่องทาง โดยเชื่อมต่อกับธนาคารและ e-Wallet ต่างๆ ในประเทศไทย

Flow การทำงานหลักของ Thai QR (TTB)

Flow A — สร้าง QR และ Polling สถานะ:
สร้าง QR Code → เพิ่มเข้าระบบ → Poll ถามสถานะจาก TTB (ทุกๆ ไม่กี่วินาที) → TTB ตอบกลับ

Flow B — ธนาคารแจ้งผลชำระเงิน (Push):
ธนาคาร TTB ส่ง Notify → บันทึกลง Database → แจ้ง Merchant → ส่ง LINE แจ้งลูกค้า

5.3 ปัญหาการเชื่อมต่อกับ Provider ภายนอก

🟠 TrueMoney Wallet — API Error (55 รายการ)
  • 400 Bad Request (38 รายการ) — TrueMoney API ตอบ error กลับมา เกิดช่วง 20:57 น. เป็นหลัก อาจเป็นปัญหาชั่วคราวฝั่ง TrueMoney
  • Payment code expired (บางส่วนจาก 17 รายการ) — ลูกค้าใช้ payment code ที่หมดอายุแล้ว
  • Insufficient fund (บางส่วนจาก 17 รายการ) — ยอดเงินในกระเป๋าไม่เพียงพอ

💡 แนวทาง: กรณี 400 Bad Request ควรตรวจสอบกับ TrueMoney ว่ามี API change หรือไม่ กรณี expired/insufficient เป็นเรื่องปกติฝั่งลูกค้า

🟠 Alipay+ — Notify Error (666 รายการ)
  • ERROR_ALIPAYPLUS_NOTIFY_TO_CUSTOMER (646 รายการ) — แจ้งผลลูกค้าไม่สำเร็จ เกิดทุกนาทีตลอดช่วง 3 ชั่วโมง
  • ALIPAYPLUS_NOTIFY_URL_ERROR02 (20 รายการ) — ไม่พบ paymentrequestid + NullReferenceException

💡 แนวทาง: ตรวจสอบ endpoint ที่ใช้ notify ลูกค้า Alipay+ ว่ายังทำงานปกติหรือไม่ และแก้ NullReference ใน code

🟠 CIL — Merchant Notify Error (1,255 รายการ)
  • ERROR_CIL_NOTIFY_URL (836 รายการ) — ไม่พบ merchant profile (merchant 200015, 200047 เป็นต้น)
  • ERROR_POST_CIL_NOTIFY_URL (414 รายการ) — POST callback ไป merchant ล้มเหลว
  • ERROR_CIL_MERCHANT_SCAN_RESPONSE_PAYMENT (5 รายการ) — NullReferenceException

💡 แนวทาง: ตรวจสอบ merchant profile ที่หายไป และ URL callback ของ merchant ว่ายังใช้งานได้

🔵 ShopeePay — Query Error (9 รายการ)

ไม่พบ transaction เมื่อ query สถานะ — อาจเป็นเพราะ transaction ยังไม่ถูกสร้างฝั่ง ShopeePay

🔵 Token Error (10 รายการ)

Token ไม่ valid สำหรับ order query — อาจเป็น token หมดอายุ ควรตรวจสอบ token refresh mechanism

6. ข้อเสนอแนะและแนวทางแก้ไข

#ปัญหาระดับแนวทางแก้ไข
1 Duplicate Key ใน order_details ด่วน เพิ่ม duplicate check (IF NOT EXISTS) ก่อน INSERT หรือใช้ MERGE statement และตรวจสอบ retry logic ที่ทำให้ส่ง transaction ซ้ำ
2 Data Truncation (String too long) ด่วน ขยาย column size ในตาราง order_details โดยเฉพาะ field ที่เก็บ Ref1, Ref2 หรือ trim ข้อมูลก่อน insert
3 Schema ไม่ตรง (savefiletransaction) ด่วน แก้ไข code ให้ใช้ชื่อ column ที่ถูกต้อง (status, mref1-8, lastupdate ไม่มีในตาราง) หรือเพิ่ม column ที่ขาด
4 CIL Merchant Profile ไม่พบ สำคัญ ตรวจสอบ merchant profile ที่หายไป (200015, 200047) และ URL callback ของ merchant
5 Alipay+ Notify ลูกค้าล้มเหลว สำคัญ ตรวจสอบ endpoint notify ลูกค้า Alipay+ และแก้ NullReferenceException ใน code
6 TrueMoney 400 Bad Request สำคัญ ประสานงานกับ TrueMoney ตรวจสอบ API specification ว่ามีการเปลี่ยนแปลงหรือไม่
7 Order Query Polling ถี่เกินไป ปรับปรุง ปรับ polling interval ให้ห่างขึ้น (เช่น exponential backoff) เพื่อลดภาระ API 36,365 queries → ลดได้มาก
8 Token Error ปรับปรุง ตรวจสอบ token refresh mechanism ให้ต่ออายุ token อัตโนมัติก่อนหมดอายุ

7. สรุปผลการวิเคราะห์

✅ จุดแข็ง

  • ความเร็วประมวลผลดีมาก — 98.2% จบใน sub-second
  • ไม่มี transaction ใดใช้เวลาเกิน 1 วินาที
  • TTB API ตอบกลับเร็ว — เฉลี่ย 0.48 วินาที
  • ระบบรองรับ peak 6 tx/sec ได้โดยไม่มีปัญหา
  • รองรับหลาย payment provider พร้อมกัน
  • มีระบบ LINE notify แจ้งลูกค้าอัตโนมัติ

⚠️ จุดที่ต้องปรับปรุง

  • DB Error 1,858 รายการ (duplicate + truncation)
  • Schema ไม่ตรงใน savefiletransaction
  • CIL notify ล้มเหลว 1,255 รายการ
  • Alipay+ notify ลูกค้าล้มเหลว 646 รายการ
  • Order Query poll ถี่เกินไป (98.4% ไม่พบ)
  • TrueMoney API error ช่วง 20:57 น.
📌 สรุปภาพรวม: ระบบ Payment Gateway ทำงานได้ดีในภาพรวม ความเร็วประมวลผลอยู่ในเกณฑ์ดีมาก ปัญหาหลักที่ต้องแก้ไขเร่งด่วนคือ Database errors (duplicate key, data truncation, schema mismatch) ซึ่งกระทบต่อการบันทึกธุรกรรมประมาณ 1,868 รายการจากทั้งหมด 7,200 รายการ (25.9%) นอกจากนี้ควรปรับปรุง notification ไปยัง merchant และลูกค้าที่ยังล้มเหลวอยู่
