🔗 3. จับคู่ SUCCESS transaction_id กับ BankRef

เปรียบเทียบ transaction_id จาก Order Query (SUCCESS) กับ BankRef จาก Bank Notify
เพื่อตรวจสอบว่ารายการที่ query สำเร็จ ตรงกับรายการที่ธนาคารแจ้งมาหรือไม่

⚠️ หมายเหตุสำคัญ: transaction_id ใน Order Query มี #suffix ต่อท้าย เช่น TMB202604122006248522#146644647091
ส่วน BankRef ใน Notify ไม่มี # เช่น TMB202604122006248522
ดังนั้นต้อง ตัด # ออก ก่อนจึงจะจับคู่ได้

📊 4. สรุปและข้อสังเกต

✅ จุดแข็ง:
  • อัตราจับคู่ SUCCESS กับ BankRef สูงถึง 97.5% — ระบบทำงานสอดคล้องกันดี
  • Bank Notify เป็นช่องทางหลัก (97.6%) ในการรับผลชำระเงิน — เชื่อถือได้
  • Dynamic QR คิดเป็น 95.3% ของ traffic — ระบบสร้าง QR ใหม่ทุกครั้งเป็นหลัก
⚠️ ข้อสังเกต:
  • 166 รายการ SUCCESS ไม่พบ BankRef — น่าจะเป็น transaction จากช่วงเวลาก่อน 20:00 น. ที่ query สำเร็จในช่วงนี้
  • STATIC QR ไม่มี match ใน Order Query — เพราะ STATIC QR ใช้ flow ที่ต่างออกไป (ไม่ผ่าน order query polling)
  • transaction_id มี #suffix — ต้องระวังเวลาเขียน code จับคู่ ต้องตัด # ออกก่อน
  • STATIC QR 336 รายการ (4.7%) — เป็นกลุ่มที่เกิด "String or binary data truncation" error ใน DB
💡 ข้อเสนอแนะ:
  • ตรวจสอบ 166 รายการที่ unmatched ว่ามี transaction จากช่วงก่อนหน้าจริงหรือไม่
  • ปรับ DB schema รองรับ STATIC QR Ref format ที่ยาวกว่าปกติ
  • พิจารณาเพิ่ม logging สำหรับ STATIC QR flow แยกต่างหาก
